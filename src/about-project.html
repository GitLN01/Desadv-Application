Introduction

This application is designed to enable users to visualize and interact with inbound DESADV data (data about goods received) extracted from CSV files. The application will provide a user-friendly interface for accessing and analyzing this information. This project consists of both frontend and a backend, with the backend responsible for consuming and manipulating the data, as well as persisting it in files or a database of choice.

On the page "Dashboard", as mentioned previously, users will be provided with a comprehensive dashboard to view and interact with inbound DESADV data. Users will be able to visualize various metrics, apply filters, and generate dynamic charts to analyze the data effectively.

Features

  • Data Visualization: The application will display the inbound DESADV data in visually appealing charts and graphs, enabling users to understand trends and patterns easily.
  • Data Interaction: Users can interact with the data by applying filters, selecting time periods, and customizing the view to suit their needs.
  • Add New CSV Files: The application will allow users to upload additional CSV files containing inbound DESADV data. The newly uploaded files will be automatically consumed and displayed in the tool without manual intervention.

Frontend and Backend functionalities

Backend Functionality Frontend Functionality
  • Data Ingestion: The backend will receive and process CSV files uploaded by users.
    		•
  • Data Visualization: Using React.js, the frontend will display the inbound DESADV data in theform of charts, graphs, and tables, making it easy for users to interpret and analyze the data.
    		•
  • Data Parsing: The CSV files will be parsed to extract relevant data, which will then be stored in the PostgreSQL database.
  • Interactive Filters: Users will be able to apply filters to view specific subsets of data, such as date ranges or specific categories.
  • Database Management: The backend will manage the database, ensuring that all data is appropriately stored and indexed for efficient retrieval.
  • Real-time Updates: The frontend will be designed to dynamically update whenever new data is received and processed by the backend.
  • Automatic Data Consumption: To automatically consume newly received CSV files, the backend will use a file-watching mechanism. When a new CSV file is detected, the backend will initiate the parsing process and update the database accordingly.
